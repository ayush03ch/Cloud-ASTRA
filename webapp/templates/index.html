üîê Role Creation Steps
üìã GUI Steps ‚ñº
  1. Open AWS Management Console and sign in
  2. Navigate to IAM ‚Üí Roles (search "IAM" in services)
  3. Click "Create Role" button
  4. Select "AWS account" as the trusted entity type
  5. In the "Account ID" field, paste your 12-digit AWS Account ID
  6. Check "Require external ID" checkbox
  7. Enter a secure external ID (e.g., CloudAstra-SecureID-2025)
  8. Click "Next", search for and select "AmazonS3FullAccess" policy
  9. Click "Next" and name the role "CloudAstraRole"
  10. Review and click "Create Role"
  11. Copy the Role ARN (format: arn:aws:iam::ACCOUNT-ID:role/CloudAstraRole)
  12. Paste both Role ARN and External ID in the form below
or
‚öôÔ∏è CLI Commands ‚ñº

1. Create Trust Policy (save as trust-policy.json):

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::YOUR-ACCOUNT-ID:root"
      },
      "Action": "sts:AssumeRole",
      "Condition": {
        "StringEquals": {
          "sts:ExternalId": "your-secure-external-id"
        }
      }
    }
  ]
}

2. Create Role (S3 example - S3FullAccess):

aws iam create-role --role-name CloudAstraRole \
  --assume-role-policy-document file://trust-policy.json

3. Attach Policy (S3):

aws iam attach-role-policy --role-name CloudAstraRole \
  --policy-arn arn:aws:iam::aws:policy/AmazonS3FullAccess
‚ö†Ô∏è Security Note: For simplicity, we use full service access policies (e.g., AmazonS3FullAccess). For your security/privacy, you can configure least privilege policies with only required permissions.
